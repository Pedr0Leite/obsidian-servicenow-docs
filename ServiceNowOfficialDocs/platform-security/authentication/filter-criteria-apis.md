# Filter criteria for APIs

[[adaptive-auth-filter-criteria|Filter criteria]] contains filter conditions or queries that are used as policy inputs for an [[c_Authentication|authentication]] policy. Policy inputs are used to group one or more filter criteria and define the policy conditions of an authentication policy. For example, an [[ip-filter|IP Filter]] Criteria define an IP address in the Classless Inter-Domain Routing \(CIDR\) format or a range of IP addresses.

You can create filter criteria based on the user's IP address, role, and the user group to which they belong.

**Note:** You also use the filter criteria created from the **[[adaptive-authentication|Adaptive Authentication]]** module. For more information, see [Adaptive authentication](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/authentication/adaptive-authentication.md).

You can create filter criteria for APIs using the same process as filter criteria for adaptive authentication. For details see [Filter criteria](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-security/authentication/adaptive-auth-filter-criteria.md).

# What is GAF?

Group Action Framework (GAF) is a platform intelligence framework for building workflows that derive insights from **sets of related data** (incidents, cases, Knowledge articles, etc.). It combines **generative AI** and **machine learning** to:

1. **Group** — cluster similar records together, label clusters, and select representative records
2. **Act** — run LLM prompts against those clusters to produce summaries, trend insights, resolution suggestions, or similarity mappings

> Plugin scope: `sn_gaf`
> 

> App type: Generative AI | Paid | First available: Xanadu (latest: v1.0.5)
> 

> Minimum release: Washington DC
>

# Core Concepts

## Two-Phase Operation

### Phase 1 — Grouping

Triggered during workflow configuration. Involves 3 skills in sequence:

| Skill | What it does | Technique |
| --- | --- | --- |
| **Grouping skill** | Clusters related records using ML techniques | Machine Learning |
| **Topic Labelling skill** | Adds human-readable names to clusters via LLM | Generative AI |
| **Action Strategy skill** | Selects representative sample records from each cluster for downstream processing | — |

Grouping can be re-triggered **manually** or on a **scheduled basis**.

### Phase 2 — Action

Triggered at runtime when a workflow needs cluster-derived insights. Involves 2 skills:

| Skill | What it does | Technique |
| --- | --- | --- |
| **Action Mapper skill** | Runs LLM inference on representative records, producing per-record summaries | Generative AI |
| **Action Reducer skill** | Aggregates mapper summaries into a single cluster-level insight (e.g. how to resolve this class of incident) | Generative AI |

# GAF vs AI Search vs Predictive Intelligence

| Feature | Purpose | Consumes Assists? | Example use cases |
| --- | --- | --- | --- |
| **GAF** | Groups similar records into clusters, then executes LLM actions on that data to derive insights on trends and commonalities | No (assists consumed only if called via AI Agent or custom skill) | Trend Analyzer AI Agent, Resolution steps generation, Knowledge Deduplication |
| **AI Search** | Retrieves semantically relevant records using contextual understanding | Only if using Now Assist Q&A genius results | Q&A in Virtual Agent / Employee Center, GAF fallback |
| **Predictive Intelligence** | Uses historical data to predict outcomes (field values, classifications) | No | Field prediction on forms, auto assignment group, record categorization |

### When to use which

- **GAF**: Large datasets (2,000–500,000 records) where you want LLM prompts run against grouped data — summarization, resolution steps generation, similarity detection
- **Predictive Intelligence**: Large datasets (30,000–300,000 records) for detecting specific repeatable patterns — routing, wait times, form field population. No GenAI, no hallucinations. Cannot feed results directly to AI Agents today.
- **AI Search**: When you want to return a specific record based on semantics

### Can GAF and AI Search be combined?

Yes. GAF has an **Optimized Result** feature (activated by default from plugin v3.0.0) that:

- Uses Predictive Intelligence ML for more efficient record clustering
- Increases the record ceiling to **500,000**
- Uses **AI Search as a fallback** when GAF cannot find a cluster match

# Configuration

## Pre-configuration checklist

Answer these before configuring GAF:

1. What problem are you trying to solve?
2. What data has the answers to your problem?
3. Do you have enough quality data to glean insights from?
4. How do you want to interact with the output of GAF?

## Data quality guidance

- **Choose fields with key terms** — e.g. `short_description` on incidents contains useful keywords; `number` field does not
- **Limit to 2–3 grouping fields** — too many fields or fields with excessive data cause incoherent clusters
- **Ensure data diversity** — a large dataset covering only one topic yields poor clustering
- **Exclude stale data** — records older than 1 year tend not to represent the current landscape (unless you lack the minimum record count)
- **Record thresholds**: minimum **500 records**, maximum ~**500,000 records**

## Grouping fields configuration

Grouping fields are stored on the `sn_nowassist_skill_config_var_set` table.

## OOTB vs Custom GAF implementations

| Scenario | Approach |
| --- | --- |
| Use OOTB GAF (e.g. ITSM Suggested Steps, Knowledge Deduplication) | Simply activate — no config needed. **Cannot be modified** at present. |
| Modify OOTB generic GAF skills | Locate skills labelled `GAF [x] grouping` in `sn_nowassist_skill_config` (where `[x]` = HR, ITSM, CSM, etc.) and modify per product docs |
| Custom table grouping | Clone a `GAF [x] grouping` skill → update table name, select fields, set filters |

# Running & Monitoring Groupings

## Manually trigger a grouping

1. Navigate to **Scheduled Script Executions** `[sysauto_script]`
2. Search for `GAF – Run Offline Flow` (specific to your cluster)
3. Open the record → click **Execute Now**

## Check scheduled run frequency

1. Navigate to `[sysauto_script]`
2. Open `GAF – Run Offline Flow`
3. Review the **Repeat Interval** field

## Check grouping progress

1. Navigate to `[sysauto_script]` → open `GAF – Run Offline Flow`
2. Scroll to the **GAF job trackers** related list
3. Also viewable directly on the `ml_solutions` table

## Grouping duration

Typically **10–30 minutes** depending on record count and field complexity.

# Key Tables

| Table | Contents |
| --- | --- |
| `sn_gaf_record_group` | Output of Grouping skill — each record = one cluster (with LLM-generated name and keywords). Contains related `sn_gaf_record_group_detail` and `sn_gaf_action_strategy_result` records. |
| `sn_gaf_record_group_detail` | Individual records belonging to each group; flags which records are cluster representatives |
| `sn_gaf_action_strategy_result` | Results of the Action Strategy skill (selected representative records per cluster) |
| `sn_gaf_action_mapper_result` | Output of the Mapper skill — maps new records to existing clusters |
| `sn_gaf_action_reducer_result` | Output of the Reducer skill — cluster-level insights (e.g. how to resolve this class of incident) |
| `sn_nowassist_skill_config` | Configuration metadata for each GAF skill (grouping, topic, action, etc.) |
| `sn_gaf_grouping_job` / `sn_gaf_job_tracker` | Execution logs for grouping jobs, including clustering pipeline metadata (sourced from `ml_solution` table) |
| `sn_gaf_action_job` | Execution logs for action strategy jobs |
| `sn_gaf_topic_job` | Execution logs for topic generation jobs |

# Consuming GAF Output in Code

GAF results are retrieved via the `GAFUtils` Script Include in the `sn_gaf` scope.

**Example — retrieve reducer summary (cluster-level insight):**

```jsx
const recordSysId   = "b7dff706c3b7ced09c9971dc7a013163";
const groupSkillId  = "11fe3ee3770412103f075cea5b5a99ee";
const actionSkillId = "d42f7535775412103f075cea5b5a994f";
const mapperSkillId = "535d2d2877e012103f075cea5b5a9970";
const reducerSkillId = "b53ead6877e012103f075cea5b5a997f";

const result = new sn_gaf.GAFUtils().getReducedSummary(
  recordSysId,
  groupSkillId,
  actionSkillId,
  mapperSkillId,
  reducerSkillId
);

gs.info("GAF Result: " + JSON.stringify(result));
```

GAF can be called from:

- AI Agents (primary consumer)
- Flow Designer flows
- Custom NAS skills (via Now Assist Skill Kit)
- Any server-side script context

# Performance & Caching

GAF uses **caching** for commonly queried clusters and records. Over time, recall improves as previously computed LLM responses are returned from cache rather than re-generated.

**Example**: A password reset incident maps to a frequently accessed cluster → cached resolution steps returned immediately.

**Optimized Prediction** (v3.0.0+, on by default):

- Uses AI Search for more efficient record recall
- Raises record ceiling to 500,000
- Introduces AI Search as fallback when GAF finds no cluster match

# Assist Consumption

GAF itself **does not consume Assists**. Assist consumption depends on the triggering mechanism:

- Called via an **AI Agent** or **custom NAS skill** → consumes Assists at that layer
- Raw GAF grouping/action jobs → no Assist consumption

Refer to the [Now Assist pricing guide](https://www.servicenow.com/content/dam/servicenow-assets/public/en-us/doc-type/legal/sn-assist-overview.pdf) or your account rep.

# Limitations & Known Constraints

| Constraint | Detail |
| --- | --- |
| **Multilingual** | Not officially supported. Processes optimized for English only; other languages may work but quality is not guaranteed. |
| **Domain Separation** | Not supported. If activated on a domain-separated instance, data will mix across domains. |
| **ACLs on records** | Behavior differs by GAF phase — check official docs for specifics |
| **OOTB modifications** | ITSM Suggested Steps and Knowledge Deduplication OOTB implementations cannot be modified at present |
| **Can a record exist in multiple groupings?** | Yes — if multiple grouping skills target the same table, a record can belong to multiple groupings |
| **Shared cluster across use cases** | Multiple AI Agents can use the same GAF cluster if their insight needs are compatible (e.g. categorization + suggested resolution both pulling from the same incident cluster) |

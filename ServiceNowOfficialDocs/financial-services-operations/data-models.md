# Data Models

Create structured and flexible data models to represent the need across various industries.

-   **[[financial-services-operations-core-data-model|Financial Services Operations Core]]**  
The [[fso-overview|Financial Services]] Operations \(FSO\) Core application provides a framework to enable financial institutions to organize and manage their data effectively to meet their business needs. This section introduces FSO Core and explains its purpose and key data design components.
-   **[[dispute-data-model|Card Disputes data model]]**  
This section outlines the card disputes data model and the tables installed with the disputes application.
-   **[[payment-card-application|Payment card]]**  
The Payment card application stores details of payment cards, which can be used across the entire card life-cycle, from issuance to servicing.
-   **[[insurance-claims-core-data-model|Insurance Claims Core]]**  
The Insurance Claims Core application includes a framework that provides generic claims data model components and approval engine that your organization can use to create flexible data structures that meet your business needs.
-   **[[financialservices-remote-tables|Financial Services Remote Tables]]**  
Integrate your core systems using the ServiceNow Financial Services Remote Tables. This feature streamlines development by eliminating the need to recreate the ServiceNow data model for integrations.

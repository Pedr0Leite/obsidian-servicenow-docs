# Add or remove a business process from a value stream stage

You can add an existing business process to a value stream stage to map your [[application-portfolio-management-landing-page|enterprise architecture]] workflows to the stages of your value streams. You can also remove associated business processes if they are no longer relevant for the value stream stage.

## Before you begin

Role required: sn\_apm.apm\_user

## Procedure

1.  Navigate to **Workspace** &gt; **[[ea-workspace|Enterprise Architecture Workspace]]**.

2.  Open the Portfolio List view by selecting the Portfolio icon \[Omitted image "portfolio-icon.png"\] Alt text: Portfolio icon.

3.  Select the expand row icon \(\[Omitted image "ExpandIcon.png"\] Alt text: Expand Row icon\) next to **Business Architecture**.

4.  Select **Value Stream Stages**.

5.  Select the value stream stage that you want to add a business process to.

    A new page appears, displaying the details of the value stream stage.

6.  Select the **Business Processes** tab.

7.  Select **Add**.

8.  On the **Add business process** pop-up, select the existing business process that you want to add to the value stream stage.

9.  Select **Add**.

    The business process is added to the value stream stage.

10. To remove an associated business process from the value stream stage, select the business process and then select **Remove**.

    The business process is removed from the value stream stage.

    **Note:**

    -   You can select and remove multiple business processes simultaneously.
    -   This action just removes the association between the value stream stage and the business process. The business process record isn’t deleted from the system.

# Set up layouts

[[layouts|Layouts]] define the structure and appearance of the configuration interface. They organize [[fields|fields]] into pages, tabs, and sections, shaping how users interact with configurations and view shopping cart details in CPQ.

A layout record defines the [[buyside-user-interface|buyside user interface]] for a configuration experience. It consists of fields and the organizational structures—pages, tabs, sections, column [[cpq-sets|sets]], and headings—that govern how fields will be represented on the screen. It also describes how the shopping cart should be displayed, by default.

When a blueprint has more than one layout defined, the user can step through them using the alternate layout \(\[Omitted image "cpq-layout-alternate-layout-button.png"\] Alt text: Save icon\) button in the upper-right corner of the buyside UI.

CPQ layouts are defined in spreadsheets and uploaded via [[matrix_loader_table_of_contents|matrix loader]]. Refer to [[csv_layout_upload|CSV layout upload]] for detailed instructions.

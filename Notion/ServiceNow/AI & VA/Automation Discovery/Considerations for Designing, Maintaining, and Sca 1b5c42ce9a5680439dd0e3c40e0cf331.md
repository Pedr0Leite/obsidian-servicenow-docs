# Considerations for Designing, Maintaining, and Scaling the Solution

## Automation Discovery for ITSM

Automation
 Discovery is a fast way to identify ITSM automation opportunities that 
can be solved with Virtual Agent. First, make sure ITSM is selected as 
the Taxonomy. This allows Automation Discovery to map incidents to 
things that can be automated in the platform. For example, all 
variations of password reset requests such as "help password is broken, 
Unable to login, Outlook password issue, Ping reset Password, MSFT AD 
reset, and Okta Reset Password" will fall under the ITSM - Request - 
Data - User - Account - Reset Password automation opportunity category.

- Automation Discovery is best utilized with Topic Recommendations to accelerate
your time to value with Virtual Agent. Use the Topic Recommendations
page to run analysis reports, view recommendations, add topics to
Virtual Agent, or link a recommendation to an existing topic.
- Topic Recommendations shows all results in this view by default. This view
lets you simultaneously run or refresh analysis reports for all
configured sources. The configured sources for the recommendations are
listed as hyperlinks. Click the link to open the settings for the
report. If a recommendation has a New status, you may select it and
click Add to Virtual Agent.
- You can view topic recommendations
by their data source. This lets you run a new analysis or refresh an
existing one for a single data source rather than running all of them
simultaneously. Click Settings to view the report's settings. If a
recommendation has a New status, you may select it and click Add to
Virtual Agent.

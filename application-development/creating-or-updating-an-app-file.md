# Creating or updating an app file with Build Agent

Use Build Agent to add new files or modify existing files in ServiceNow applications to assist with ongoing metadata and app development and maintenance.

The process for creating or updating an app file is very similar to creating an application. The main difference is that [[build-agent|Build Agent]] prompts you to enter the application name or scope so it knows which application to update files in.

In a new chat with Build Agent, select **Create a file** or **Update a file** to get started, and continue the conversation from there.

\[Omitted image "sn-studio-ba-create-update-file.png"\] Alt text: Use the Create a file and Update a file options to get started adding files to your application.

Continue the conversation just like you did with creating an application, using the change log and checkpoints as you go. For more information, see [[ba-conversational-change-log|Build Agent conversation change log]] and [[revert-app-changes-using-build-agent|Revert app changes with Build Agent]].

Review the application and its metadata in [[servicenow-studio-landing|ServiceNow Studio]] using the change log. For more information, see [Build Agent conversation change log](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/ba-conversational-change-log.md). You can also review the application using the File Navigator in ServiceNow Studio. For more information, see [Find an app or app file using the Navigator panel](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/servicenow-studio-classic/qs-find-app-app-file-using-navigator-panel.md).

For information on deploying your application, see [[build-agent-deployment|Deploying what you built with Build Agent]].

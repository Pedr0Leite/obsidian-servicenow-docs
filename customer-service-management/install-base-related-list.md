# Related list for an install base item

In the [[c_CustomerServiceManagement|Customer Service Management]] application, you can use the Install base related list to track information that is related to the install base hierarchy, cases, entitlements, and other entities. By using these tables, you can understand your customers' purchases and provide efficient post-sales support.

<table id="table_hdg_jxt_fwb"><thead><tr><th>

Related list

</th><th>

Description

</th></tr></thead><tbody><tr><td>

[[child-install-base-item|Create a child install base item]]

</td><td>

List of all the child [[install-base-item|install base items]] associated to the parent install base item.

</td></tr><tr><td>

[[sold-product|Sold Products]]

</td><td>

List of all the products or services sold to a customer. For more information on how to create a sold product, see [[create-sold-item|Create a sold product]].

</td></tr><tr><td>

Cases

</td><td>

List of all the cases associated to the current install base item. The list of cases enables customer service agents to access all the cases and issues that are related to an install base in one place. For more information on cases and case access, see [[industry-data-model-cases|Service Model Foundation cases]].

</td></tr><tr><td>

All Cases

</td><td>

List of all the child cases associated to the parent install base item. **Note:** You cannot see the parent cases that are associated with a child install base item.

</td></tr><tr><td>

[[create-entitlement|Create an entitlement]]

</td><td>

List of the existing cases for an account, product, or service contract that defines the type of support that a customer receives. For more information on entitlements, see [[configure-csm-entitlements|Configure entitlements]].

</td></tr><tr><td>

[[edit-contract|Edit a contract]]

</td><td>

List of all the contracts associated to the current install base item.

</td></tr><tr><td>

Install Base Related Parties

</td><td>

List of all the additional parties that have access to an install base item. You can add related parties only to the root install base item. For more information to the list of related parties and default responsibilities, see [[adding-related-party-config-to-case|Create related party configurations]].

 To add a related party to an install base, see [[add-related-party-install-base|Add related parties to an install base item]].

</td></tr></tbody>
</table>

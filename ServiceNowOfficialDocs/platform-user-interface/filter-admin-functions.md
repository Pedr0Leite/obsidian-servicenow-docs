# Filter admin functions

Administrators can work with [[c_Filters|filters]] in the Filters table \[sys\_filter\], including creating, editing, and deleting filters.

Administrators can also create scripted filters that cannot be created in the [[c_ConditionBuilder|condition builder]]. Scripted filters required a knowledge of JavaScript. Scripted filters can be used to create additional [[r_UsingTheDynamicOperator|dynamic filter options]].

-   **[[t_EditingSavedFilters|Create and edit filters]]**  
Set up and edit an admin-configured filter.
-   **[[t_DeletingFilters|Delete filters]]**  
You can delete any saved filter, including global, group, or personal filters.
-   **[[t_ScriptedFilters|Create scripted filters]]**  
The condition builder alone cannot create some filters, such as displaying a record set that depends on an unrelated table. If you know JavaScript, you can create JavaScript functions for use in advanced filters.
-   **[[t_DynamicFilterOptions|Create a dynamic filter option]]**  
Dynamic filter options enhance filtering by allowing users to run existing script includes or JavaScript against a [[onboarding-modals-reference|reference]] field within condition builders and dynamic reference qualifiers.

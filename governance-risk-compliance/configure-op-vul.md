# Setting up the Operational vulnerability module

Configure the [[exploring-op-vul|Operational vulnerability]] feature in the [[grc-opres-landing-page|Operational Resilience]] Workspace to help identify and address operational vulnerabilities within your organization.

## Operational vulnerability configuration overview

The Operational vulnerability feature is included by default with the Operational Resilience application.

Here are the typical configuration steps in the Operational Resilience Workspace:

1.  Report an operational vulnerability by creating an Operational vulnerability record.
2.  Review the assigned and unassigned Operational vulnerability records. The Operational Resilience administrator \[sn\_oper\_res.admin\] can assign the Operational vulnerability record.
3.  Review the status of the approvals for the assigned operational vulnerabilities.
4.  [[set-up-op-vul-type|Set up the Operational vulnerability type]].
5.  [[establish-state-model-record|Set up the State model and Action task model]] to define the state transitions for the operational vulnerability record and action tasks.
6.  [[create-vul-assmt-template|Set up the Vulnerability Assessment Template]] for configuring important assessment details such as the assessment duration, associated tables, filter conditions of the operational vulnerability.
7.  [[create-doc-template|Set up the Document Template]] \(HTML Template\) for generating the PDF of the operational vulnerability.

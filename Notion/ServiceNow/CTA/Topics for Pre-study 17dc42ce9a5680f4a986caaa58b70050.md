### Additional information about the CTA:

A case study each week

One of the challenges is the Technical Governance

In the end there is a capstone project where you have to deliver the whole solution to the “customer”

They ask tricky questions

You have 2 weeks to prepare for the final exam

There are 2 workshops, one in the morning and one in the afternoon so all the people can present.

Each student need to review at least 2 presentations and give constructive feedback

Put some effort on making the slides with as little text as possible and talk about the topics

The CTA should be valid for eternity, at least for now.

I think it will have a maintain course or something like a delta for that.

Tips on how to maintain the platform

- The arch needs to think about manageability, performance, security and upgradability.

Other question during the program:

- “We need to enable omnichannel interaction for our customers” - You need to suggest the various options, that makes sense, for example, VA, SP, Email integration, mobile app.
- Be aware that some solutions might have basic requirements, for example, predictive inteligence might need a certain amount of data.

### Tips for the capstone presentation:

Cover all the topics in my scenario

There is no right or wrong solution, you need to know how to justify all your choices for your solutions.

30mins max! You can go beyond this, so do the presentation for like 28min

Some of my bullets points should be about capability maps, architecture standards, domain separation (if you want or think it makes sense/recommended), instance terá-duplication(?), 

You will have 10/11 topics that you need to cover, they might be evaluated from 0 to 3.

- If you mention the topic
- If you can explain this topic
- If you are good at this topic

Everything that you might recommend, be prepared to explain why and have a solid knowledge.

No technical details, because it will consume a lot of time. You are talking to a “client”, so its better to explain the solution rather than the technical side.

A type of question might be “Ok, we don’t have money for the Module X (or for example, Governance), what can you do?”

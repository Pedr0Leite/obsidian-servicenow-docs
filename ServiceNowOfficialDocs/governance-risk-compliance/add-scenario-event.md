# Adding a scenario event to the analysis

You can add a scenario event to the [[scenario-analysis-ov|scenario analysis]] and analyze its impact on your business service. If you are the scenario analysis owner, you can add the stakeholders and reviewers to the scenario analysis in [[grc-opres-landing-page|Operational Resilience]] Workspace. You can then collate the observations of all participants and use their inputs to analyze the scenario.

## Workflows for adding a scenario event

You can add a scenario event to the scenario analysis by completing one of the following workflows:

-   You can add a scenario event to the scenario analysis first and then add the participants, dependencies, services, issues, and so on. For more information on this workflow, see [[add-a-scenario-event|Add a scenario event]].
-   You can also add a participant to the scenario analysis first and then add the scenario event, services, issues, and so on. For more information on this workflow, see [[add-a-participant-to-the-scenario-event|Add a participant and monitor the responses]].

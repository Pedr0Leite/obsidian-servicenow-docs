# For PPT

### **Slide 1: Title Slide**

**Title:** Week 9 – Integration Leading Practices, Platform Data, and Integration Strategy

**Subtitle:** Sovereign Bank Case Study – Certified Technical Architect Program

**Presented to:** Li Na, Enterprise Architect

---

### **Slide 2: Recap & Objective**

- **Recap:**
    
    Last week, we created a Service Portal and GenAI strategy to support Sovereign Bank’s self-service goals and agent productivity.
    
- **This Week’s Objective:**
    
    Plan integration strategies, clarify foundational data governance, and determine how ServiceNow fits into Sovereign’s enterprise architecture.
    

---

### **Slide 3: Desired Business Outcomes**

- Automate integration with HR systems and Slack
- Enable secure SAML 2.0 authentication
- Maintain centralized access via Azure AD or equivalent
- Provide a clear, non-technical integration roadmap
- Simplify integration delivery for internal teams

---

### **Slide 4: Integration Challenges at Sovereign Bank**

- Avoiding a disruptive “big bang” rollout
- Leadership misalignment → decision delays
- Unclear integration landscape → user frustration
- IP/firewall limitations → technical blockers
- Complex IT perception → change fatigue
- Multiple ESBs → lack of standardization

---

### **Slide 5: Integration Prioritization Strategy**

**Approach:**

1. **Start with high-value, low-risk integrations**:
    - Slack for employee support
    - HR system sync for foundational data
2. **Create an Integration Value Matrix** → score based on impact vs. effort
3. **Establish an Executive Integration Steering Committee**
    - Align priorities and ensure buy-in
4. **Release iteratively by function (ITSM, HR, CSM)**

---

### **Slide 6: Supporting the Bank’s IT Teams**

- **Enablement through documentation & training**
    - IntegrationHub spoke templates
    - Prebuilt use cases for key systems
- **Define a central intake process for integration requests**
- **Create reusable Integration Patterns**
    - E.g., “Slack Notification Flow,” “Employee Onboarding HR Feed”
- **Designate Integration Champions** in each business unit

---

### **Slide 7: Foundational Data & Governance**

**Foundational Elements:**

- User records, group memberships, roles
- Location, department, cost centers
- CMDB relationships and CI classification

**Governance Strategy:**

- Centralized Data Owner model
- Regular audits & automated validation rules
- Define Source of Truth (HR, AD) for key data elements
- Sync schedules and error-handling rules

---

### **Slide 8: Integration Best Practices**

- **Minimize point-to-point custom code** → use IntegrationHub, MID Servers
- **Use data transform maps and flow logic to decouple**
- **Preserve platform performance** with asynchronous patterns
- **Leverage scoped apps to isolate integrations**
- **Utilize REST over SOAP unless legacy requires otherwise**

---

### **Slide 9: ESB & Firewall Considerations**

- **TIBCO Integration:**
    - Use the HTTP/S protocol with MID Server mediation
    - Secure traffic with mTLS (mutual auth)
    - Collaborate on payload structure, response validation
- **Firewalls/IPs:**
    - Document firewall rules and IP whitelist needs
    - Schedule early environment access tests

---

### **Slide 10: Recommended Integration Patterns**

- **Inbound REST API for ticket creation** (e.g., from Slack)
- **Outbound data sync via IntegrationHub (HR → User table)**
- **Virtual Agent integration for Slack & MS Teams**
- **Batch-based FTP/SFTP data pull for legacy systems**
- **Use Event-based Integration for Alert Management**

---

### **Slide 11: Additional Systems for Integration**

- Azure AD / Identity Management
- SAP HR / Workday
- Slack / MS Teams
- Salesforce / CRM tools
- Banking platforms (Risk & Fraud systems)
- Data Lakes or Reporting Tools (e.g., Power BI)

---

### **Slide 12: Conclusion & Next Steps**

- Build phased roadmap and executive alignment on integration sequence

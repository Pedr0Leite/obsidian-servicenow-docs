# Service Portal configuration page widgets

[[c_ServicePortal|Service Portal]] uses widgets for configuration. When you [[configure-onboarding-modals|configure]] a page or a widget for a portal in the [[service-portal-configuration-page|Service Portal Configuration page]], you actually use Service Portal widgets to do so.

Service Portal configuration widgets are internal only and cannot be extended.

Configuration page widgets are included as part of the Service Portal Configuration Pages plugin \[com.glide.service-portal.config\], which is activated automatically when you activate the Service Portal for Enterprise Service Management plugin \[com.glide.service-portal.esm\].

Navigate to **Service Portal** &gt; **Service Portal Configuration** to view these widgets in action.

-   **[Service Portal Config Overview widget](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/service-portal/config-overview-widget.md)**  
The [[config-overview-widget|Service Portal Config Overview widget]] is the widget that appears on the sp\_config homepage. Use this widget to navigate Service Portal Configuration. You can use this base system widget as-is in your portal or clone it to suit your own business needs.
-   **[Portal config widget](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/service-portal/portal-config-widget.md)**  
The [[portal-config-widget|Portal Config widget]] is the left panel of the Branding Editor, which you use to configure themes for your portal. You can use this base system widget as-is in your portal or clone it to suit your own business needs.
-   **[SP Page Map widget](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/service-portal/page-map-widget.md)**  
The Page Map widget works in the Page Editor to display any Service Portal page in a tree structure. You can use this base system widget as-is in your portal or clone it to suit your own business needs.
-   **[SP Portal Map widget](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/service-portal/portal-map-widget.md)**  
The Portal Map widget displays your portal in a tree structure.
-   **[Theme Preview widget](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/service-portal/theme-preview-widget.md)**  
The [[theme-preview-widget|Theme Preview widget]] appears in the Branding Editor and allows you to see any theme changes you make to your portal. You can use this base system widget as-is in your portal or clone it to suit your own business needs.
-   **[Widget Edit Panel widget](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/service-portal/edit-panel-widget.md)**  
The Widget Edit Panel appears on the main page of the Widget Editor in Service Portal Configuration \(**All** &gt; **Service Portal** &gt; **Service Portal Configuration** &gt; **Widget Editor**\). You can use this base system widget as-is in your portal or clone it to suit your own business needs.

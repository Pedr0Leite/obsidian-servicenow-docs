# Functions of ServiceNow Extensions for VS Code

The ServiceNow Extensions for VS Code extension includes tools for developing on the ServiceNow AI Platform.

The extension adds several functions to your Visual Studio Code \(VS Code\) implementation. You access these functions through the command palette.

\[Omitted image "vscode-syncfiles.png"\] Alt text: Visual Studio Code command palette

<table id="table_s33_ftn_w3b"><thead><tr><th>

Functionality

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Setup Now Workspace

</td><td>

Create the project folder to work with ServiceNow applications. For details on creating this workspace, see [[setup-workspace|Set up a workspace in VS Code]].

</td></tr><tr><td>

Activate Now Extension

</td><td>

Activates the ServiceNow extension. See [[activate-servicenow-ext|Activate ServiceNow Extensions for VS Code]].

</td></tr><tr><td>

Create Now Project

</td><td>

Import existing ServiceNow applications to a Visual Studio Code project. See [[create-project|Create a project in VS Code]].

</td></tr><tr><td>

Sync Current Project

</td><td>

During development, synchronize all files between VS Code and the instance. See [[synchronize-files|Synchronize the current project between a Visual Studio Code workspace and a ServiceNow instance]].

</td></tr><tr><td>

Sync Current File

</td><td>

Synchronize the current file you are working on. See [[sync-current-file|Sync the current file between a Visual Studio Code workspace and a ServiceNow instance]].

</td></tr><tr><td>

Clear Credentials

</td><td>

Deletes the entered credentials in the Settings page. See [[clear-credentials|Clear instance credentials in Visual Studio Code]].

</td></tr><tr><td>

Reset Project

</td><td>

Sets project to its original state. See [[reset-project|Reset a project in Visual Studio Code]].

</td></tr><tr><td>

Configure File Types

</td><td>

Modify the options selected in the metadata. Metadata determines which elements of your application you want to synchronize with VS Code. See [[vscode-import-application|Import an application into Visual Studio Code]].

</td></tr><tr><td>

Add Custom File Types

</td><td>

Add file types to your project using the Custom File Types wizard. See [[vscode-add-custom-filetypes|Add custom file types in Visual Studio Code]].

</td></tr><tr><td>

Select Application

</td><td>

Switch between application within the workspace in VS code. See [Import an application into Visual Studio Code](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/vscode-import-application.md).

</td></tr><tr><td>

Select Update Set

</td><td>

Select an update set. Changes synchronized to your instance are applied to the selected update set. For more information on update sets, see [Import an application into Visual Studio Code](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/application-development/vscode-import-application.md).

</td></tr><tr><td>

Create New File

</td><td>

Create records in your application. For details, see [[vscode-create-file|Create a file in VS Code]].

</td></tr><tr><td>

Global File Search

</td><td>

Find files within the instance. See [[search-file|Search files on your instance in VS Code]].

</td></tr><tr><td>

Run Background Scripts

</td><td>

Run a background script on your instance. See [[vscode-background-script|Run background scripts using VS Code]].

</td></tr><tr><td>

IntelliSense

</td><td>

An IntelliSense code-completion aid available for Glide APIs. See [[Intellisense|IntelliSense in VS Code]].

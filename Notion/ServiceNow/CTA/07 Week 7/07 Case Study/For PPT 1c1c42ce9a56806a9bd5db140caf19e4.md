# For PPT

### **Week 7: User Experience, Information Architecture, and Interface Design**

### **Key Factors & Challenges**

- **Custom Code Overuse** → Developers resist existing ServiceNow libraries, slowing implementation.
- **Lack of User Testing** → Releases go live without validation, causing usability issues.
- **Complex Application Ecosystem** → 421+ business applications create navigation and integration challenges.
- **Scalability Issues** → 80,000+ employees need an intuitive system, replacing a heavily customized Remedy ITSM.

---

### **Discussion Points & Solutions**

✅ **Technical Scripting & Developer Adaptation**

- Developers are likely familiar with **Remedy's AR scripting** or **legacy JavaScript frameworks**.
- Train them on **ServiceNow’s Glide API**, **Flow Designer**, and **Next Experience UI Builder** to streamline development.

✅ **Information Architecture: Mapping 421+ Systems**

- Use **Application Portfolio Management (APM)** to inventory all systems.
- Categorize applications based on **business capability mapping** and usage frequency.
- Implement **CMDB with Business Service Mapping** for logical structuring.

✅ **Categorization & Organization Strategy**

- **Hybrid Scheme** → Mix exact (structured categories like departments) and subjective (frequent actions) classifications.
- **Hierarchical Structure** → Organize content by role-based access, ensuring users see relevant systems.

✅ **Addressing Key Impediments**

- **Custom Code Resistance** → Educate developers on **low-code/no-code benefits** for faster deployment.
- **Testing Gaps** → Implement **user feedback loops & A/B testing** before full rollout.
- **Navigation Complexity** → Use **Next Experience's Unified Navigation** to consolidate key services.

✅ **Next Experience Components**

- **Unified Navigation** → Centralized, role-based workspace for efficiency.
- **UI Builder** → Drag-and-drop interfaces without excessive coding.
- **Playbooks & Workspaces** → Pre-configured UI layouts for common workflows.

---

### **Presentation Recommendations**

🎯 **1. Risks of Ignoring UX**

- Low adoption, increased training costs, and inefficiencies.
- Propose **continuous UX validation** (design thinking workshops, beta testing).

🎯 **2. Organizational System Schemes**

- **Combination of Exact & Subjective Schemes** → Exact for structured processes, subjective for user convenience.

🎯 **3. Information Structure**

- **Role-Based & Task-Oriented Navigation** → Prioritize high-usage functions per user group.

🎯 **4. Architectural Support for Labeling, Navigation, and Search**

- **Consistent Metadata & Taxonomy** → Standardize terminology across all applications.
- **Auto-Suggest Search & AI-Powered Filters** → Improve discoverability.
- **Breadcrumb Navigation & Context-Aware Menus** → Guide users efficiently.

By implementing these strategies, Sovereign Bank can create a seamless, **user-friendly** ServiceNow experience while ensuring **scalability and efficiency**. 🚀
